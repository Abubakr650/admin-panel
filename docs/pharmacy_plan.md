# خطة تنفيذ نظام الصيدلية (Pharmacy Module Implementation Plan)

هذه الخطة توضح مراحل بناء واجهات وبرمجيات قسم الصيدلية في لوحة تحكم العيادة.

## 1. الكنترولر (Controllers)
سنحتاج إلى الكنترولر التالية لإدارة العمليات:
- `PharmacyItemController`: لإدارة الأصناف الدوائية (إضافة، تعديل، عرض).
- `PharmacyBatchController`: لإدارة الدفعات (Batches) وتواريخ الانتهاء والكميات الواردة.
- `SupplierController`: لإدارة الموردين والوكلاء.

## 2. المسارات (Routes)
تعريف مسارات (Resource Routes) لكل قسم:
- `/admin/pharmacy/items`
- `/admin/pharmacy/batches`
- `/admin/pharmacy/suppliers`

## 3. الواجهات (Views - Blade Templates)
تصميم واجهات احترافية وسهلة الاستخدام باستخدام أنماط حديثة:
- **قائمة الأصناف (Items List)**: جدول يعرض الأدوية مع البحث والفلترة حسب النوع أو التصنيف.
- **إضافة صنف (Create Item)**: نموذج يتضمن كافة البيانات (الاسم التجاري، العلمي، الشكل الدوائي، ملاحظات).
- **إدارة المخزون (Inventory/Batches)**: واجهة خاصة لمتابعة الكميات المتبقية لكل دفعة مع نظام تنبيه لتواريخ الانتهاء القريبة.

## مراحل التنفيذ المقترحة:

### المرحلة الأولى: التأسيس وبرمجة الموردين
1. إنشاء `SupplierController`.
2. إنشاء واجهات الموردين (إضافة/عرض).
3. ربط الموردين بقاعدة البيانات.

### المرحلة الثانية: إدارة الأصناف الدوائية
1. إنشاء `PharmacyItemController`.
2. إنشاء واجهات الأصناف (قائمة الأصناف، نموذج الإضافة).
3. تنفيذ عمليات الـ CRUD الأساسية.

### المرحلة الثالثة: إدارة الدفعات والمخزون (الأهم)
1. إنشاء `PharmacyBatchController`.
2. إنشاء واجهة لإدخال الكميات الجديدة (Batches) وربطها بالأصناف والموردين.
3. تصميم واجهة "حالة المخزون" التي تظهر الكميات الكلية وتنبيهات الأدوية المنتهية.
