# نظام إدارة عيادة الأسنان — وصف شامل للنظام

> هذا النص مخصص لإرساله لنموذج ذكاء اصطناعي آخر كـ context كامل عن النظام.

---

## نظرة عامة

نظام إدارة عيادة أسنان متكامل مبني بـ **Laravel** (PHP) في الخلفية و **Tailwind CSS** في الواجهة.
يعمل النظام على بيئة ويب ويتضمن لوحة تحكم للموظفين والأطباء، وموقعاً إلكترونياً للمرضى للحجز عبر الإنترنت.

---

## الوحدات الأساسية

### 1. وحدة العيادة (Clinic)

- **المرضى**: تسجيل بيانات المريض (الاسم، تاريخ الميلاد، الجنس، رقم الهاتف، الصورة). يُحسب العمر تلقائياً من تاريخ الميلاد.
- **الأطباء**: إدارة ملفات الأطباء مع صور البروفايل والتخصصات.
- **المواعيد**: جدولة المواعيد مع حالات (قادم / منجز / ملغي)، وربط كل موعد بمريض وطبيب.

### 2. وحدة الصيدلية (Pharmacy)

- **الأصناف (Items)**: إدارة الأدوية والمستلزمات الطبية مع دعم باركود QR للبحث والإضافة السريعة.
- **الدُفعات (Batches)**: كل دواء له دفعات تحتوي على تاريخ الإنتاج وتاريخ انتهاء الصلاحية والكمية.
- **المستودع (Warehouse)**: مخزون المواد الخام والمستلزمات.
- **الموردون (Suppliers)**: بيانات الموردين وتاريخ التوريد والعقود.
- **تنبيهات انتهاء الصلاحية**: يعمل Laravel Scheduler يومياً ويفحص جدول `pharmacy_batches`، فإذا كان أي دواء سينتهي خلال 30 يوماً أو أقل يُرسل إشعاراً للمسؤول ويُلوّن الصنف في الواجهة (أصفر = قريب من الانتهاء، أحمر = انتهت الصلاحية).

### 3. وحدة الفوترة (Billing)

- **الفواتير**: إنشاء فواتير مفصّلة لكل زيارة مع بنود متعددة.
- **المدفوعات**: تتبع المبالغ المدفوعة والمتبقية.
- **العملات**: دعم متعدد العملات مع جداول أسعار الصرف.
- **الخدمات**: قائمة خدمات قابلة للتخصيص مع الأسعار.

### 4. وحدة الأشعة (Radiology)

- **طلبات الأشعة**: ربط كل طلب بمريض وطبيب.
- **صور الأشعة**: رفع صور الأشعة (X-Ray) وتخزينها على Cloudinary.
- **التحليل بالذكاء الاصطناعي**: عند رفع صورة الأشعة، يُرسلها النظام إلى **Google Gemini** عبر API، ويستقبل تقريراً تحليلياً يوضح احتمالية وجود تسوس أو انتانات أو مشاكل في الجذور أو العظام. يظهر التقرير بجانب صورة الأشعة في الواجهة. الطبيب يراجع التقرير ويتخذ القرار النهائي.
- **التقارير**: تخزين نتائج التحليل وملاحظات الطبيب.

### 5. وحدة تقويم الأسنان (Orthodontics)

إدارة حالات التقويم طويلة المدى، متابعة خطة العلاج، وتسجيل جلسات التقويم المتعددة.

### 6. وحدات الدعم

- **المصادقة (Auth)**: تسجيل الدخول، إدارة المستخدمين، الصلاحيات والأدوار.
- **الإعدادات**: إعدادات النظام والعيادة.
- **الثيمات**: 5 ثيمات ملونة قابلة للتبديل في الواجهة عبر `ThemeService`.
- **تخزين الملفات**: دعم Cloudinary لرفع الصور وتخزينها.

## الموقع الإلكتروني للحجز (Patient Booking Portal)

موقع منفصل مرتبط بنفس قاعدة البيانات، يتيح للمرضى:

- عرض معلومات العيادة والخدمات والأطباء.
- اختيار طبيب وموعد متاح وحجزه مباشرةً.
- إدخال بياناتهم (الاسم، رقم الهاتف) بدون الحاجة لإنشاء حساب.
- استقبال تأكيد فوري.

عند إتمام الحجز، يُنشأ موعد جديد في لوحة التحكم ويُرسل تأكيد للمريض عبر واتساب.

---

## نظام رسائل واتساب

النظام يُرسل رسائل واتساب تلقائية للمرضى في الحالات التالية:

| الحالة | محتوى الرسالة |
|--------|---------------|
| تأكيد الموعد | اسم الطبيب، التاريخ، الوقت، العنوان |
| تذكير قبل 24 ساعة | تذكير بتفاصيل الموعد القادم |
| نتيجة الأشعة جاهزة | إشعار بأن التقرير متاح |
| فاتورة جديدة | رابط لعرض الفاتورة ومبلغها |

يعتمد الإرسال على **WhatsApp Business Cloud API** (Meta) عبر `WhatsAppService` مرتبط بـ `AppointmentController`.

## البنية التقنية

| المكوّن | التقنية |
|---------|---------|
| الخلفية (Backend) | Laravel (PHP) |
| الواجهة (Frontend) | Blade Templates + Tailwind CSS + Alpine.js |
| قاعدة البيانات | MySQL |
| تخزين الصور | Cloudinary |
| الذكاء الاصطناعي | Google Gemini API |
| رسائل الواتساب | WhatsApp Business Cloud API (Meta) |
| المهام الدورية | Laravel Scheduler |
