2.1.1	Introduction about Digital Library:
This is a Library System that was developed using C# and ASP.NET with SQL Server 2008 as the backend database. It provides basic functionalities for managing library resources but lacks several key features found in more modern systems.
Hosting online was the most problematic one. The application allows librarians to add books to the library inventory but does not provide the capability to delete books. Furthermore, the system suffers from stability issues and lacks role-based access control, potentially compromising security and data integrity.


Fig 2.1- Login page of Digital Library system
2.1.2	Disadvantages of Digital Library:
1.	Framework Dependent: C# inherently is not a web development language, hence the pathway to learn web development with it is longer.
2.	Licensing Costs: Microsoft SQL Server often requires licensing fees, especially for enterprise-level deployments.
3.	Learning Curve: C# may have a steeper learning curve, especially for beginners or developers from non-Microsoft backgrounds.
4.	Limited Cross-Platform Support: SQL Server is primarily optimized for Windows environments, as other alternatives are supported in multiple operating systems.
 
5.	Design Inconsistency: The Design of Digital Library is not very consistent, as its structure is not responsive.
6.	Lack of Security: This Library Management System does not include any security measures nor validations for checking, and making sure to only receive the right data.

1. Fun Design: The design of Digital library is vibrant, colorful, so in perspective, it can get repurposed to be a valid option.
 
2.2.1	Introduction about Online Library Management System:
This is a Library System that was developed PHP and MySQL as the backend database. It provides much more advanced features compared to the earlier project. Here, hosting online is not a problem, as some hosting websites allow PHP and MySQL website apps to be hosted for free. This website allows librarians to add books to the library inventory, monitor the issued books, and also capable of handling further tasks like full book operations (add, update and delete) and issued book monitoring.


Fig 2.3- Admin Dashboard of Online Library Management System
2.2.2	Advantage of Online Library Management System:
1.	Better Design: This Design is much better in comparison to the earlier depicted system, as it is responsive and also well-themed
2.	More Functionalities: The Online Library Management System has more functionalities in perspective than the earlier system, such as managing issued books, complete book operations, and book issuance and also monitoring them.
3.	More Secure: The Online Library Management System uses an encryption for the user and admin passwords, whereas the Digital Library does not use any encryption in perspective.
 


 
Fig 2.4 – Student Dashboard of Online Library Management System
2.2.3	Disadvantages of Online Library Management System:
1.	No Book Quantity Handling: Although the Online Library Management System does indeed expand on the earlier system, it does not handle the quantity of the books a library has meaning that for either book it sells, it is either infinite, or has only one copy of each book.
2.	Outdated Encryption Used: As in perspective, the Online Library Management System does include an encryption method, it applies the MD5 Encryption, which an outdated hash encryption that is able to be cracked and exploited.
3.	No Support for other languages: As capable as it is, it lacks the support for other languages across the system.
4.	No optimization for uploads: As this system, we found out that the files uploaded (such as images) are stored as it is, meaning no compression for fast storing and loading.
